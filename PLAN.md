# 세계사 주요 사건 타임라인 웹 프로젝트 계획서

> 첫 화면의 승인된 카드 디자인을 수정하거나 새 카드를 추가할 때는 반드시 `DESIGN_GUIDE.md`를 함께 확인한다. 사진+그레이스케일 역사 지도 합성 구조와 전쟁별 이미지 파일, 정확한 CSS 기준값은 해당 문서를 따른다.

## 1. 프로젝트 목표

세계사의 굵직한 사건을 사건별 인터랙티브 타임라인으로 보여주는 웹사이트를 만든다. 첫 구현 범위는 다음 두 화면이다.

1. 사건을 선택하는 첫 페이지
2. 제2차 세계대전 상세 타임라인 페이지

향후 제1차 세계대전, 6·25 전쟁 등의 타임라인도 동일한 구조로 추가할 수 있어야 한다.

이 문서는 구현 코드가 아니라, 추후 Claude AI로 구현할 때 기준으로 사용할 제품·화면·데이터·인터랙션 계획서다.

---

## 2. 핵심 사용자 경험

사용자가 사이트에 들어오면 먼저 대표 이미지와 제목으로 구성된 사건 카드를 본다. 카드를 선택하면 해당 사건의 상세 페이지로 이동한다.

상세 페이지에서는 시간은 위에서 아래로 흐르고, 지역은 왼쪽에서 오른쪽으로 배치된다. 각 지역에는 굵은 세로 타임라인이 있으며, 전투·선언·침공·회담 등의 사건이 해당 지역 선에 연결된다. 서로 다른 전선에 영향을 준 사건은 지역 간 화살표로 연결한다.

사용자가 상세 페이지에서 바로 이해해야 할 내용은 다음과 같다.

- 지금 보고 있는 시점이 언제인지
- 같은 시기에 각 지역에서 무슨 일이 벌어졌는지
- 한 지역의 사건이 다른 지역에 어떤 영향을 주었는지
- 전쟁의 흐름이 어느 시점에 바뀌었는지

---

## 3. 첫 구현 범위

### 포함

- 메인 사건 선택 페이지
- 제2차 세계대전 상세 페이지
- 데스크톱 및 모바일 반응형 화면
- 연도·중요 월 기준의 가로 시간 구분선
- 지역별 세로 타임라인
- 사건 포인트와 요약 카드
- 지역 간 인과·지원·병력 이동 화살표
- 사건 상세 내용 열기
- 범례와 기본 필터
- 향후 사건을 추가하기 위한 공통 데이터 구조

### 초기 버전에서 제외

- 회원 가입 및 로그인
- 사용자 작성·수정 기능
- 지도 기반 애니메이션
- 실시간 협업
- 다국어 지원
- 관리자 페이지
- 지나치게 많은 소규모 전투 수록

초기 목표는 모든 사건을 망라하는 것이 아니라, 제2차 세계대전의 전체 흐름과 전선 간 상호작용을 명확히 전달하는 것이다.

## 4. 정보 구조와 화면 경로

권장 경로는 다음과 같다.

- `/`: 세계사 사건 선택 페이지
- `/events/world-war-2`: 제2차 세계대전 상세 페이지
- 향후 `/events/world-war-1`
- 향후 `/events/korean-war`

각 사건은 공통 상세 화면을 사용하고, 사건별 데이터만 교체하는 구조로 설계한다.

## 5. 첫 페이지 계획

### 5.1 화면 목적

사용자가 보고 싶은 역사적 사건을 이미지와 제목으로 빠르게 선택하게 한다.

### 5.2 구성

1. 상단 영역
   - 사이트 제목: 예) `세계사 타임라인`
   - 짧은 설명: `동시에 벌어진 사건과 전선 간 영향을 한눈에 살펴보세요.`
   - 초기에는 복잡한 메뉴를 두지 않는다.

2. 사건 카드 그리드
   - 대표 이미지
   - 사건명
   - 기간
   - 한 줄 소개
   - 카드 전체를 클릭 가능하게 만든다.

   - 카드는 중요도나 제작 순서가 아니라 역사적 시작 시점이 오래된 순서로 정렬한다.
   - 기원전 연도부터 현대까지 같은 정렬 규칙을 사용한다.
   - 전쟁뿐 아니라 혁명, 제국의 성립·붕괴, 종교·문화 확산, 탐험, 조약, 경제위기 등 굵직한 역사 사건도 같은 카드로 등록할 수 있게 한다.
   - 전쟁과 일반 사건을 구분하는 `주제 유형`을 표시하되 첫 화면의 시간 순서는 유지한다.

3. 초기 카드
   - 제2차 세계대전: 활성 카드, 상세 페이지로 이동
   - 제1차 세계대전: `준비 중` 상태
   - 6·25 전쟁: `준비 중` 상태

### 5.3 카드 예시

| 사건 | 기간 | 이미지 방향 | 상태 |
|---|---|---|---|
| 제2차 세계대전 | 1939–1945 | 여러 전선을 상징하는 역사 사진 또는 지도풍 이미지 | 활성 |
| 제1차 세계대전 | 1914–1918 | 참호와 병사 | 준비 중 |
| 6·25 전쟁 | 1950–1953 | 한반도 지도 또는 피란 행렬 | 준비 중 |

이미지는 저작권이 명확한 퍼블릭 도메인·공개 라이선스 자료를 사용하고, 출처와 라이선스를 기록한다. 대표 이미지 위에는 제목을 읽기 쉽도록 어두운 오버레이를 적용한다.

### 5.4 상호작용

- 카드에 마우스를 올리면 이미지 확대, 명도 변화, 테두리 강조 중 1~2개만 사용한다.
- 키보드 Tab으로 카드 선택이 가능해야 한다.
- 활성 카드는 클릭 시 상세 페이지로 이동한다.
- 준비 중 카드는 이동시키지 않고 상태를 명확히 표시한다.

### 5.5 첫 페이지 공통 주제 데이터

첫 화면의 각 카드는 다음 공통 정보를 가진다.

- `id`
- `title`
- `startDate`: 정렬 기준
- `endDate` 선택값
- `displayPeriod`
- `topicType`: war, revolution, empire, politics, culture, religion, exploration, economy, disaster, other
- `summary`
- `coverImage`
- `coverImageAlt`
- `status`: draft, upcoming, published, archived
- `detailPath`

기원전 날짜는 내부적으로 음수 연도 또는 시대와 연도를 분리한 정렬값을 사용한다. 화면에는 `기원전 218년`처럼 읽기 쉬운 문자열을 별도로 표시한다. 시작일이 불확실한 고대 사건에는 정렬용 대표 연도와 `연도 추정` 표시를 함께 저장한다.

## 6. 제2차 세계대전 상세 페이지 개념

### 6.1 기본 축

- 세로축: 시간, 위에서 아래로 1939년에서 1945년까지 진행
- 가로축: 지역 또는 전선
- 지역별 중심에는 굵은 세로선을 배치
- 연도 및 중요한 월에는 화면 전체를 가로지르는 구분선을 배치
- 사건은 해당 지역의 세로선에 포인트로 연결
- 타 지역에 직접 영향을 준 사건은 포인트 사이 화살표로 연결

### 6.2 권장 지역 열

초기에는 다음 4개 열을 권장한다.

1. 유럽
2. 지중해·북아프리카
3. 동부전선
4. 아시아·태평양

이 구분은 `유럽·아프리카·태평양`의 단순 3열보다 제2차 세계대전의 핵심 흐름을 설명하기 쉽다. 특히 독소전쟁의 사건 밀도가 높으므로 동부전선을 분리해야 서유럽 사건과 겹침이 줄어든다. 지중해와 북아프리카는 작전상 강하게 연결되어 하나의 열로 묶는다.

필요하면 추후 다음과 같이 확장할 수 있다.

- 대서양 전선 별도 분리
- 중국 전선 별도 분리
- 외교·본국 정치 전용 열 추가

### 6.3 지역 헤더

각 세로선 상단에는 고정 또는 반고정 헤더를 둔다.

- 지역명
- 대표 색상
- 간단한 아이콘 또는 축약 표식
- 현재 필터 적용 여부

스크롤 중에도 어느 열인지 놓치지 않도록 지역 헤더를 상단에 고정하는 방식을 우선 검토한다.

## 7. 시간 구분선 계획

### 7.1 기본 규칙

- 모든 연도 시작점에 굵은 가로선과 큰 연도 라벨을 표시한다.
- 전쟁 흐름이 크게 바뀌는 중요한 월에는 중간 굵기의 가로선을 표시한다.
- 일반 월은 별도 선 없이 사건 카드의 날짜로 표시한다.
- 모든 달을 같은 간격으로 표시하면 페이지가 지나치게 길어지므로, 초기 버전은 사건 중심의 비선형 간격을 사용한다.

### 7.2 비선형 시간축에 대한 표시

사건이 많은 기간은 넓게, 적은 기간은 좁게 배치하되 사용자가 실제 시간 간격으로 오해하지 않도록 다음 중 하나를 적용한다.

- 화면 상단 범례에 `사건 밀도에 따라 세로 간격이 조정됨` 표기
- 연도 사이에 물결 모양 생략 표시 사용
- 중요 월 라벨을 명시해 위치 판단을 돕기

정확한 시간 비율보다 사건 간 관계를 중시하는 교육용 타임라인으로 정의한다.

### 7.3 우선 강조할 시점

- 1939년 9월: 폴란드 침공, 유럽 전쟁 시작
- 1940년 5~6월: 서유럽 전역과 프랑스 함락
- 1941년 6월: 바르바로사 작전
- 1941년 12월: 진주만 공격과 미국 참전
- 1942년 하반기: 엘 알라메인·스탈린그라드·과달카날
- 1943년 7~9월: 시칠리아 침공과 이탈리아 항복
- 1944년 6월: 노르망디 상륙
- 1945년 5월: 독일 항복
- 1945년 8~9월: 원자폭탄, 소련의 대일 참전, 일본 항복

## 8. 사건 표현 방식

### 8.1 사건 포인트

각 사건은 지역 세로선 위 또는 가까이에 원형 포인트로 표시한다.

- 기본 사건: 작은 원
- 전쟁 흐름을 바꾼 핵심 사건: 큰 원 또는 이중 테두리
- 여러 달에 걸친 작전: 시작·종료 포인트를 잇는 세로 구간
- 결과가 확정된 시점: 종료 포인트에 별도 표식

### 8.2 사건 카드

포인트에서 짧은 가로 연결선을 뻗어 사건 카드를 배치한다. 카드는 같은 열 안에서 좌우로 번갈아 배치하거나, 각 지역 열의 정해진 카드 영역에 배치한다.

카드의 기본 정보는 다음과 같다.

- 날짜 또는 기간
- 사건명
- 1~2문장 요약
- 사건 유형 아이콘
- 교전국 또는 주요 행위자
- 중요도
- 상세 열기 버튼 또는 카드 전체 클릭

### 8.3 상세 내용

사건 카드를 클릭하면 데스크톱에서는 우측 패널, 모바일에서는 전체 폭의 바텀시트 또는 별도 상세 영역을 연다.

상세 정보는 다음 순서로 구성한다.

1. 사건명과 날짜
2. 배경
3. 전개
4. 결과
5. 다른 전선에 미친 영향
6. 관련 인물·국가
7. 출처

초기 버전에서는 상세 페이지 안에 중첩된 별도 URL을 만들기보다 패널을 우선 사용하되, 공유 필요성이 확인되면 사건별 앵커 또는 쿼리 파라미터를 추가한다.

### 8.4 사건 유형 색상

지역 색상과 사건 유형 색상을 동시에 많이 사용하면 혼란스러울 수 있다. 따라서 세로선·헤더는 지역 색상, 사건 카드의 작은 배지나 아이콘만 사건 유형 색상을 사용한다.

권장 사건 유형은 다음과 같다.

- 전투·작전
- 침공·점령
- 외교·회담·조약
- 참전·항복
- 정치·정책
- 민간인 피해·전쟁범죄
- 기술·군수·정보전

## 9. 지역 간 화살표 계획

### 9.1 화살표의 의미

화살표는 단순히 같은 시기에 일어난 사건이 아니라, 자료로 설명 가능한 직접 영향만 표시한다.

예시:

- 북아프리카 승리 → 연합군의 시칠리아 침공
- 동부전선의 압박 → 독일군 전력 분산 → 서유럽 방어력에 영향
- 진주만 공격 → 미국의 참전 → 유럽과 태평양 양쪽의 전쟁 역량 확대
- 노르망디 상륙 → 독일의 양면전선 압박 심화
- 독일 항복 → 연합군 자원의 태평양 전선 이동

### 9.2 시각 규칙

- 기본은 얇은 곡선 화살표
- 출발 포인트와 도착 포인트를 명확히 연결
- 마우스를 올리거나 선택했을 때만 진하게 강조
- 선택하지 않은 화살표는 낮은 불투명도로 표시
- 화살표 중앙 또는 툴팁에 관계 유형을 표시

관계 유형 예시:

- 병력·장비 이동
- 전략적 영향
- 참전 또는 외교적 결과
- 연합 작전
- 대응 작전

### 9.3 혼잡 방지

- 기본 화면에는 가장 중요한 연결만 노출한다.
- `전선 간 영향 보기` 토글로 모든 화살표를 숨기거나 표시할 수 있게 한다.
- 특정 사건을 선택하면 그 사건과 연결된 화살표만 강조한다.
- 교차가 많은 경우 화살표를 화면 가장자리 또는 열 사이의 전용 통로로 우회시킨다.
- 한 사건에 연결이 많으면 개별 선 대신 중간 집결 노드를 검토한다.

## 12. 반응형 화면 계획

### 데스크톱

- 4개 지역 열을 동시에 비교하는 화면을 기본으로 한다.
- 최소 콘텐츠 폭을 정하고 화면이 좁으면 가로 스크롤을 허용한다.
- 연도 라벨은 왼쪽 고정 영역에 표시한다.
- 상세 패널은 오른쪽에서 연다.

### 태블릿

- 2~3개 열만 보이게 하고 가로 스크롤을 사용한다.
- 지역 헤더를 고정해 현재 열을 확인할 수 있게 한다.

### 모바일

4개 열을 한 화면에 축소하면 읽기 어려우므로 다음 방식을 권장한다.

- 상단 탭으로 지역을 하나씩 선택
- 선택 지역의 세로 타임라인을 전체 폭으로 표시
- 다른 전선과의 연결은 화살표 대신 `영향을 준 사건`·`영향을 받은 사건` 링크 카드로 표시
- `전체 전선 비교`는 선택 기능으로 제공하고 가로 스크롤 허용
- 상세 내용은 바텀시트 또는 별도 전체 화면 패널로 표시

즉, 데스크톱은 동시성 비교, 모바일은 지역별 탐색에 우선순위를 둔다.

## 14. 시각 디자인 원칙

- 역사 자료를 다루는 차분하고 신뢰감 있는 인상을 사용한다.
- 배경은 따뜻한 회백색 또는 매우 짙은 남색 중 하나로 정하고, 카드와 선의 대비를 충분히 확보한다.
- 지역 색은 색각 이상 사용자를 고려해 명도와 선 패턴도 함께 구분한다.
- 본문 글자 크기는 모바일 기준 최소 16px 수준을 유지한다.
- 날짜, 제목, 설명의 시각적 위계를 명확히 한다.
- 국기만으로 국가를 표현하지 않고 텍스트를 병기한다.
- 역사 사진에는 대체 텍스트와 출처를 제공한다.
- 화려한 애니메이션보다 읽기와 관계 파악을 우선한다.

## 15. 접근성 계획

- 모든 카드와 컨트롤을 키보드로 조작 가능하게 한다.
- 포커스 위치를 명확히 표시한다.
- 색상만으로 지역·사건 유형·중요도를 구분하지 않는다.
- 이미지에 의미 있는 대체 텍스트를 작성한다.
- 화살표 관계는 화면 읽기 프로그램에서도 `A 사건이 B 사건에 영향을 줌` 형태의 텍스트로 제공한다.
- 모션 감소 설정 사용자는 스크롤·패널 애니메이션을 최소화한다.
- 상세 패널이 열리면 포커스를 패널로 이동하고, 닫을 때 원래 카드로 돌려보낸다.
- 확대 200%에서도 핵심 콘텐츠가 유실되지 않게 한다.

## 16. 역사 콘텐츠와 출처 관리

### 원칙

- 사건 설명은 최소 2개 이상의 신뢰 가능한 자료로 교차 검토한다.
- 날짜와 명칭이 자료마다 다르면 선택 근거를 기록한다.
- 국가 중심의 승패 서술만 하지 않고 군인·민간인 피해를 구분한다.
- 논쟁적인 표현은 단정하지 말고 학계의 일반적 견해와 쟁점을 구분한다.
- 이미지마다 제작자, 연도, 소장처, 라이선스, 원본 URL을 기록한다.

### 권장 출처 범주

- 국가 기록원·박물관·공공 아카이브
- 대학 출판부와 학술 서적
- 신뢰 가능한 역사 백과사전
- 1차 사료는 맥락 설명과 함께 사용

Wikipedia는 사건 탐색과 참고문헌 발견에는 사용할 수 있으나, 최종 설명의 유일한 근거로 삼지 않는다.

## 17. 기술 구현 방향

구현 도구는 Claude AI로 코딩을 시작할 때 최종 선택하되 다음 조건을 충족해야 한다.

- 사건별 공통 레이아웃 재사용
- 정적 데이터 기반으로 빠른 초기 로딩
- 관계 화살표를 곡선으로 그릴 수 있는 SVG 레이어
- 카드 위치 변화에 맞춰 화살표 재계산
- 반응형 레이아웃
- 검색 엔진이 사건 제목과 설명을 읽을 수 있는 구조
- 이미지 최적화
- 자동화 테스트 가능

권장 구현 개념:

- 페이지 UI: 컴포넌트 기반 프레임워크
- 타임라인 선과 화살표: HTML 카드 + SVG 오버레이 조합
- 콘텐츠: 초기에는 TypeScript 또는 JSON 데이터, 내용이 커지면 Markdown/MDX 또는 CMS 검토
- 카드 위치 측정: DOM 위치를 읽어 SVG 시작·종료 좌표 계산
- 큰 화면의 긴 타임라인: 필요할 때만 렌더링 최적화 검토

Canvas보다 SVG를 우선 검토한다. SVG는 화살표 선택, 접근성용 라벨, 반응형 좌표 조정, 디버깅이 상대적으로 쉽다.

## 19. 구현 순서

### 1단계: 프로젝트 골격

- 프레임워크와 스타일링 방식 결정
- `/`와 `/events/world-war-2` 경로 생성
- 공통 레이아웃·색상·타이포그래피 정의
- 데이터 스키마 초안 작성

완료 기준: 두 경로를 이동할 수 있고 임시 데이터가 표시된다.

### 2단계: 첫 페이지

- 사건 카드 그리드 구현
- 제2차 세계대전 대표 카드 활성화
- 제1차 세계대전·6·25 전쟁 준비 중 카드 추가
- 반응형 및 키보드 조작 확인

완료 기준: 이미지나 제목을 포함한 카드 전체를 눌러 제2차 세계대전 페이지로 이동한다.

### 3단계: 타임라인 정적 레이아웃

- 4개 지역 열 구성
- 지역별 세로선 구현
- 연도·중요 월 가로선 구현
- 8~10개의 시험 사건 배치
- 데스크톱 가로 정렬과 모바일 지역 탭 구현

완료 기준: 화살표 없이도 시간과 지역에 따른 사건 흐름이 읽힌다.

### 4단계: 사건 카드와 상세 패널

- 카드 요약 정보 구현
- 중요도와 사건 유형 표현
- 상세 패널 구현
- URL 앵커 필요 여부 검토

완료 기준: 사건을 선택해 배경·전개·결과·출처를 확인하고 닫을 수 있다.

### 5단계: 지역 간 관계

- 관계 데이터 구조 확정
- SVG 화살표 레이어 구현
- 선택·강조·툴팁 구현
- 화면 크기 변경 시 좌표 재계산
- 화살표 숨기기 토글 구현

완료 기준: 핵심 연결 5개 이상이 정확한 포인트 사이에 표시되고 카드와 겹침이 심하지 않다.

### 6단계: 콘텐츠 확장

- 제2차 세계대전 핵심 사건 25~35개 입력
- 설명과 출처 교차 검토
- 이미지 라이선스 검토
- 민감 사건 서술 검수

완료 기준: 1939~1945년의 큰 흐름과 각 전선의 전환점이 연결되어 보인다.

### 7단계: 품질 점검

- 모바일·태블릿·데스크톱 테스트
- 키보드·스크린리더 기본 점검
- 화살표 겹침과 카드 충돌 수정
- 느린 이미지 로딩과 레이아웃 흔들림 개선
- 주요 브라우저 확인

완료 기준: 핵심 사용 흐름에 치명적 오류가 없고, 모바일에서 내용을 읽을 수 있다.

## 20. 테스트 계획

### 기능 테스트

- 첫 페이지에서 제2차 세계대전 카드로 이동하는가
- 브라우저 뒤로 가기로 첫 페이지에 돌아오는가
- 연도 이동과 필터가 올바르게 작동하는가
- 사건 상세 패널이 열리고 닫히는가
- 화살표 토글과 선택 강조가 작동하는가
- 창 크기 변경 후 화살표가 올바른 포인트를 가리키는가

### 콘텐츠 테스트

- 날짜순 정렬이 맞는가
- 사건이 올바른 지역에 배치됐는가
- 관계의 방향과 설명이 역사적 인과를 과장하지 않는가
- 사건마다 출처가 연결됐는가
- 이미지 출처와 라이선스가 기록됐는가

### 시각 테스트

- 긴 제목이 카드 밖으로 넘치지 않는가
- 같은 시기의 카드가 서로 겹치지 않는가
- 화살표가 사건 제목을 가리지 않는가
- 연도 가로선이 모든 지역에 일관되게 이어지는가
- 모바일에서 가로 스크롤 여부가 명확한가

## 21. 주요 위험과 대응

### 사건이 많아 화면이 복잡해짐

- 중요도 1~3 단계 적용
- 기본은 핵심 사건만 표시
- 상세 사건 보기 기능을 별도로 제공

### 화살표가 지나치게 많이 겹침

- 기본 연결 수 제한
- 사건 선택 시 관련 선만 강조
- 열 사이 전용 통로와 곡선 경로 사용

### 모바일에서 다중 전선 비교가 어려움

- 모바일은 지역 탭을 기본값으로 사용
- 화살표 관계를 텍스트 링크 카드로 대체

### 시간 간격이 왜곡되어 오해 발생

- 비선형 축임을 명시
- 연도와 중요 월 라벨을 충분히 제공

### 역사적 정확성과 관점 논란

- 출처를 화면에 공개
- 중요한 서술은 교차 검토
- 논쟁적 사안은 쟁점을 분리해서 설명

## 26. 시점별 지역·세계 세력 지도

### 26.1 기능 목표

사용자가 타임라인의 특정 연도, 월 또는 사건을 선택하면 그 시점의 전선과 세력 분포를 지도에서 확인할 수 있게 한다. 지도는 타임라인을 대체하는 화면이 아니라, 사건의 공간적 맥락을 설명하는 보조 화면으로 사용한다.

지도 보기 방식은 다음 두 종류로 구분한다.

- `지역 지도`: 선택한 사건 주변의 국가, 도시, 전선, 진격 방향을 자세히 표시
- `전체 세력 지도`: 같은 시점의 연합국, 추축국, 중립국, 점령지 및 주요 전선을 세계 또는 대륙 단위로 표시

### 26.2 권장 사용자 흐름

1. 사용자가 연도 가로선, 중요 월 또는 사건 카드를 선택한다.
2. `이 시점 지도 보기` 버튼을 누른다.
3. 데스크톱에서는 큰 지도 패널 또는 모달이 열린다.
4. 모바일에서는 전체 화면 지도로 전환한다.
5. 지도 상단에서 `지역 / 전체` 보기를 전환한다.
6. 이전·다음 시점 버튼 또는 시간 슬라이더로 전쟁의 변화를 비교한다.
7. 지도 안의 사건 표식을 선택하면 해당 타임라인 사건으로 이동한다.

타임라인을 스크롤할 때 지도가 매 순간 자동 변경되면 산만하고 성능 부담이 생길 수 있으므로, 초기 버전은 사용자가 명시적으로 지도를 여는 방식을 권장한다. 추후 화면이 충분히 안정되면 지도 패널을 고정하고 타임라인 선택과 동기화하는 보기를 추가한다.

### 26.3 지도에서 표시할 정보

- 당시의 국가 및 점령지 경계
- 연합국, 추축국, 중립국 구분
- 점령지와 괴뢰국은 본국과 다른 패턴으로 구분
- 주요 전선선
- 공격 및 철수 방향
- 주요 도시, 항구, 해협
- 선택 사건의 위치
- 해전은 해역 중심점과 작전 범위
- 사건 기간 중 전선이 변하면 시작·종료 경계를 비교
- 범례, 기준 날짜, 자료 출처

모든 국가를 현대 국경으로 칠한 뒤 당시 세력만 덧씌우면 역사적으로 부정확할 수 있다. 현재 지형 데이터는 바탕 지도에만 사용하고, 당시 국경·점령지·전선은 별도의 역사 데이터 레이어로 관리한다.

### 26.4 권장 기준 시점

모든 날짜의 지도를 처음부터 만들지 않고 전쟁 흐름이 크게 바뀐 스냅샷부터 제작한다.

1. 1939년 9월: 폴란드 침공
2. 1940년 6월: 프랑스 함락 직후
3. 1941년 6월: 바르바로사 작전 개시
4. 1941년 12월: 미국 참전과 태평양 전쟁 확대
5. 1942년 11월: 추축국 최대 팽창기 전후
6. 1943년 2월: 스탈린그라드 전투 종료
7. 1943년 9월: 이탈리아 항복
8. 1944년 6월: 노르망디 상륙과 바그라티온 작전 시기
9. 1945년 5월: 독일 항복
10. 1945년 8월: 소련의 대일 참전과 일본 항복

사건에 정확히 대응하는 지역 지도는 위 스냅샷과 별도로 추가한다. 예를 들어 노르망디 상륙작전은 프랑스 북부 확대 지도, 미드웨이 해전은 중부 태평양 확대 지도를 제공한다.

### 26.5 지도 확보 및 제작 방식

권장 방식은 `기존의 검증된 역사 지도 우선 사용 + 필요한 경우에만 재가공 또는 신규 제작`이다. 유명 전쟁의 주요 시점은 이미 군사학교, 도서관, 기록원, 박물관, 역사 지도집 등에 지도가 존재하므로 같은 내용을 처음부터 다시 그리지 않는다.

우선순위는 다음과 같다.

1. 공공기관·군사학교·대학·박물관의 기존 지도 중 웹 사용이 가능한 자료
2. 공개 도메인 또는 재사용이 허용된 Wikimedia Commons 지도
3. 출처가 신뢰할 만하고 사용 허가를 받을 수 있는 지도
4. 적절한 기존 지도가 없거나 웹 가독성이 낮을 때만 직접 단순화한 SVG 지도 제작

기존 지도를 사용할 때는 다음 두 방식을 구분한다.

- `원본 지도 보기`: 기존 지도 이미지를 그대로 보여주고 확대·축소 기능 제공
- `웹 요약 지도`: 원본 지도를 근거로 핵심 세력, 전선, 이동 경로만 단순화해 별도 제작

초기 버전에서는 원본 지도 보기를 기본으로 하여 제작 시간을 줄인다. 지도 위에는 웹사이트 자체 레이어로 선택 사건의 위치, 간단한 강조 표시, 설명 버튼 정도만 추가한다. 원본 지도 내용을 임의로 변경하지 않는다.

지도마다 해상도, 색상, 범례, 투영법이 달라 화면 스타일이 통일되지 않을 수 있다. 이를 억지로 통일하기보다 공통 프레임, 제목, 날짜, 출처 영역을 동일하게 만들고 지도 자체는 원본의 역사적 자료성을 유지한다.

직접 제작이 필요한 경우에만 다음 조합을 사용한다.

- 해안선, 강, 지형의 기본 형태: Natural Earth 같은 공개 지도 데이터
- 당시 국경, 점령지, 전선: 기존 역사 지도를 교차 검토해 단순화
- 출력 형식: 반응형 SVG 또는 GeoJSON 기반 웹 지도
- 공격 화살표와 사건 표식: 별도 작전 레이어

Natural Earth의 현재 국경 데이터는 과거 정치 경계를 그대로 제공하지 않는다. 직접 제작할 때도 바탕 지형으로만 사용하고 역사 경계는 기존 역사 자료를 근거로 작성한다.

### 26.6 AI 이미지 사용 원칙

AI로 생성한 지도를 실제 역사 세력 지도로 사용하지 않는다. 생성형 AI는 국경, 지명, 날짜, 부대 위치를 틀리게 표현할 위험이 크기 때문이다.

AI 사용 가능 범위:

- 첫 페이지의 분위기용 대표 이미지
- 지도 배경의 종이 질감
- 역사적 사실을 표현하지 않는 장식 요소

AI 사용 금지 또는 수동 검증 필수 범위:

- 당시 국경과 점령지
- 부대 위치와 이동 경로
- 전선 변화
- 지명과 날짜
- 세력 관계

AI로 시각 스타일을 만든 경우에도 실제 선과 텍스트는 검증된 데이터로 다시 올린다. 지도에는 `AI 생성 지도`라는 모호한 표현 대신 사용 데이터와 편집 출처를 명확히 표시한다.

### 26.7 다운로드 자료 사용 원칙

다음 출처를 우선 검토한다.

- Natural Earth: 공개 도메인 해안선·지형·기본 벡터 지도
- 미국 의회도서관의 제2차 세계대전 군사 상황 지도: 특히 1944~1945년 서부전선 검증
- Wikimedia Commons: 파일별 라이선스와 원출처를 확인한 역사 지도
- 국가 기록원, 전쟁박물관, 공공 아카이브의 공개 지도

다운로드한 지도는 각 파일의 라이선스를 개별 확인한다. 검색 결과의 미리보기나 출처 불명 이미지를 그대로 사용하지 않는다. 파일마다 다음 메타데이터를 저장한다.

- 지도 제목
- 제작자 및 제작 기관
- 제작 연도
- 원본 URL
- 라이선스
- 접근 날짜
- 지도에서 참고한 요소
- 수정 또는 벡터화 여부

기존 지도를 발견했더라도 다음 조건 중 하나에 해당하면 사이트에 직접 저장하거나 표시하지 않는다.

- 저작권 또는 재사용 조건을 확인할 수 없음
- 교육 목적 열람만 허용되고 웹 재배포는 허용되지 않음
- 워터마크 제거 또는 무단 편집이 필요함
- 원출처가 아닌 블로그·SNS 재업로드본만 존재함

이 경우에는 원본 페이지로 연결하거나, 별도로 사용 허가를 받거나, 공개 자료를 근거로 새 요약 지도를 제작한다.

### 26.8 지도 데이터 구조

각 지도 스냅샷은 다음 정보를 가진다.

- `id`
- `title`
- `effectiveDate`
- `endDate` 선택값
- `scope`: world, region, battle
- `regionId` 선택값
- `projection`
- `viewport`
- `territories`
- `frontLines`
- `operations`
- `locations`
- `relatedEventIds`
- `sourceIds`
- `accuracyNote`

세력 영역 데이터에는 다음을 기록한다.

- `geometryId`
- `controller`
- `legalOwner` 선택값
- `status`: homeland, occupied, puppet, disputed, neutral
- `validFrom`
- `validTo`
- `sourceIds`

`controller`와 `legalOwner`를 분리해 점령 상태를 단순한 영토 소유로 오해하지 않게 한다.

### 26.9 화면 구성

지도 패널 상단:

- 지도 제목과 기준 날짜
- 지역·전체 보기 전환
- 이전·다음 스냅샷
- 시간 슬라이더
- 닫기 버튼

지도 본문:

- 세력 및 점령지 색상
- 전선선과 작전 화살표
- 선택 사건 강조
- 확대·축소 및 이동

지도 하단:

- 세력·점령 상태 범례
- `이 시점의 주요 사건` 목록
- 정확도 및 경계 설명
- 자료 출처

### 26.10 모바일 대응

- 지도는 전체 화면으로 연다.
- 손가락 확대·축소와 이동을 지원한다.
- 복잡한 범례는 접을 수 있게 한다.
- 전체 세계 지도에서는 작은 지명을 줄이고 중요한 표식만 표시한다.
- 지도 아래에 해당 시점의 사건 목록을 제공한다.
- 지도의 사건을 선택하면 닫힌 뒤 원래 타임라인 위치로 돌아가게 한다.

### 26.11 구현 단계 추가

지도 기능은 기본 타임라인의 레이아웃이 승인된 뒤 진행한다.

1. 기준 시점과 사건별 기존 지도 후보 조사
2. 지도별 출처·라이선스·해상도 기록
3. 1942년 11월 전후 지도 한 장으로 원본 지도 뷰어 검증
4. 타임라인 사건과 지도 패널 연결
5. 기준 시점 10개의 기존 지도 확보 및 등록
6. 핵심 사건 5개의 지역 상세 지도 확보 및 등록
7. 적절한 지도가 없는 항목만 웹 요약 지도로 제작
8. 출처와 라이선스 화면 추가
9. 역사적 경계와 전선 위치를 별도 검수

### 26.12 지도 기능 1차 완료 기준

- 타임라인의 중요 시점에서 `지도 보기`를 실행할 수 있다.
- 전체 세력 지도와 지역 지도를 전환할 수 있다.
- 기준 날짜, 세력, 점령 상태, 출처가 표시된다.
- 기존 공개 지도를 중심으로 최소 10개의 전체 지도 스냅샷이 제공된다.
- 기존 공개 지도를 중심으로 최소 5개의 핵심 사건에 지역 상세 지도가 제공된다.
- 지도 표식과 타임라인 사건 사이를 양방향으로 이동할 수 있다.
- AI 생성물이 역사적 사실 데이터로 사용되지 않는다.
- 다운로드 자료의 라이선스와 수정 내역이 기록된다.

## 27. 타임라인·지도 콘텐츠 편집 기능

### 27.1 기능 목표

관리자가 웹 화면에서 특정 시점의 사건 포인트, 기간선, 지역 간 관계선, 설명, 세력 지도를 직접 추가·수정·삭제할 수 있게 한다. 제2차 세계대전뿐 아니라 이후 추가되는 모든 역사 사건이 같은 편집 기능을 사용한다.

편집 대상은 다음과 같다.

- 단일 시점 사건 포인트
- 시작일과 종료일이 있는 기간·작전선
- 사건과 사건을 잇는 영향 관계선
- 사건 제목, 날짜, 요약, 상세 설명
- 관련 국가·인물·사건 유형·중요도
- 특정 시점의 전체 세력 지도
- 특정 사건의 지역 상세 지도
- 지도 위의 위치 표식, 이동 화살표, 전선선, 설명
- 출처, 이미지 출처, 라이선스 정보

### 27.2 편집 모드

일반 방문자의 열람 화면과 관리자 편집 화면을 분리한다.

- 보기 모드: 선택, 상세 열기, 필터, 지도 확인만 가능
- 편집 모드: 추가, 위치 조정, 연결, 수정, 삭제 가능
- 편집 모드는 로그인한 관리자에게만 표시
- 편집 중임을 상단 배너와 다른 배경색으로 명확히 표시
- 저장되지 않은 변경이 있으면 페이지 이탈 전에 경고

초기 개발 단계에서 로그인 기능을 아직 만들지 않는다면 로컬 개발 환경에서만 편집 모드를 활성화한다. 공개 사이트에 무방비한 편집 API를 노출하지 않는다.

### 27.3 우클릭 컨텍스트 메뉴

타임라인 편집 모드에서는 마우스 우클릭으로 커스텀 컨텍스트 메뉴를 열 수 있다.

빈 타임라인 영역 우클릭:

- 이 시점에 사건 추가
- 기간 사건 시작
- 시간 구분선 추가
- 이 시점의 세력 지도 추가
- 붙여넣기

기존 사건 포인트 또는 카드 우클릭:

- 상세 내용 편집
- 복제
- 다른 사건과 연결
- 지역 이동
- 지도 연결 또는 교체
- 삭제

관계선 우클릭:

- 관계 설명 편집
- 관계 유형 변경
- 시작·도착 사건 변경
- 삭제

지도 우클릭:

- 위치 표식 추가
- 작전 화살표 추가
- 전선선 그리기
- 설명 라벨 추가
- 해당 위치에 사건 연결

### 27.4 브라우저·운영체제 메뉴와의 충돌 처리

웹에서는 `contextmenu` 이벤트를 처리하고 기본 동작을 막으면 커스텀 메뉴를 표시할 수 있다. 브라우저나 운영체제 우클릭 메뉴와 동시에 겹쳐 뜨지 않도록 다음 규칙을 사용한다.

- 편집 모드의 타임라인·지도 캔버스 내부에서만 기본 우클릭 메뉴를 차단
- 입력창, 링크, 이미지 출처, 일반 본문에서는 브라우저 기본 메뉴 유지
- 보기 모드에서는 기본 우클릭 메뉴를 차단하지 않음
- 커스텀 메뉴에 `브라우저 메뉴 열기`가 꼭 필요하지는 않지만, 원본 지도 링크와 다운로드 기능은 별도 버튼으로 제공
- 메뉴 바깥 클릭, Escape, 스크롤 시 메뉴 닫기
- 화면 가장자리에서 메뉴가 잘리지 않도록 위치 자동 조정

macOS의 Control+클릭과 트랙패드 보조 클릭도 브라우저에서 같은 컨텍스트 메뉴 이벤트로 처리할 수 있다. 브라우저 확장 프로그램이나 일부 보조 기술이 우클릭 동작을 사용할 수 있으므로, 우클릭은 편리한 단축 동작으로만 사용하고 유일한 편집 진입 방법으로 만들지 않는다.

### 27.5 우클릭 이외의 필수 진입 방식

- 편집 도구 모음의 `+ 추가` 버튼
- 사건 카드의 `⋯` 메뉴
- 선택 후 나타나는 편집 패널
- 키보드의 Shift+F10 또는 Context Menu 키
- 포커스된 항목에서 Enter로 상세 편집
- 모바일에서는 길게 누르기, 선택 후 `⋯` 버튼 또는 하단 편집 도구 모음

모바일의 길게 누르기는 커스텀 컨텍스트 메뉴를 여는 빠른 진입 방식으로 지원한다. 단, 브라우저의 텍스트 선택·이미지 저장 메뉴와 충돌하지 않도록 편집 모드의 타임라인·지도 캔버스와 편집 가능한 항목에서만 적용한다.

길게 누르기 규칙:

- 약 500~700ms 동안 누르면 메뉴 열기
- 누르는 동안 작은 진행 표시 또는 진동 피드백 제공
- 손가락이 일정 거리 이상 움직이면 길게 누르기를 취소하고 스크롤로 처리
- 빈 영역에서는 `사건·기간·지도 추가` 메뉴 표시
- 기존 항목에서는 `편집·복제·연결·삭제` 메뉴 표시
- 메뉴가 열린 뒤 손가락을 떼어도 유지
- 시스템의 텍스트 선택이 필요한 설명 본문과 입력창에는 적용하지 않음

더블클릭·더블탭도 선택적으로 지원할 수 있지만 핵심 기능의 유일한 진입 방법으로 사용하지 않는다.

- 데스크톱 더블클릭: 기존 사건의 빠른 편집 열기
- 모바일 더블탭: 브라우저 확대, 지도 확대 동작과 충돌할 수 있어 기본값에서는 사용하지 않음
- 지도에서 더블탭 확대를 유지하고 편집은 길게 누르기로 구분
- 사용자 테스트 후 충돌이 없을 때만 더블탭 빠른 편집 옵션 검토

권장 최종 조작 체계는 `데스크톱 우클릭`, `모바일 길게 누르기`, `모든 환경의 +·⋯ 버튼`이다.

### 27.6 추가 작업 흐름

빈 영역에서 사건을 추가하는 흐름:

1. 원하는 지역 열과 시간 위치에서 우클릭 또는 `+ 추가`
2. 시스템이 해당 위치에서 지역과 대략적인 날짜를 자동 추정
3. 추가 유형 선택: 포인트, 기간, 구분선, 지도
4. 편집 폼에서 정확한 날짜와 내용을 입력
5. 출처 및 중요도 입력
6. 미리보기 확인
7. 저장

마우스 좌표로 추정한 날짜는 임시값일 뿐이며 저장 전에 날짜 입력값을 반드시 확인한다. 비선형 시간축에서는 좌표와 실제 날짜가 정확히 비례하지 않기 때문이다.

### 27.7 관계선 추가 작업 흐름

1. 출발 사건의 메뉴에서 `다른 사건과 연결` 선택
2. 출발 사건을 강조하고 연결 모드 진입
3. 도착 사건 선택
4. 관계 유형, 방향, 라벨, 상세 설명 입력
5. 미리보기에서 화살표 경로 확인
6. 저장

잘못된 자기 연결, 존재하지 않는 사건 연결, 중복 관계는 저장 전에 경고한다. 양방향 관계가 필요하면 단방향 두 개가 아니라 별도 양방향 유형으로 관리한다.

### 27.8 지도 추가·편집 작업 흐름

1. 기준 날짜 또는 관련 사건 선택
2. `세력 지도 추가` 선택
3. 공개 지도 파일 업로드 또는 원본 URL 등록
4. 지도 제목, 기준 날짜, 범위, 출처, 라이선스 입력
5. 필요하면 지도 위에 사건 포인트·전선·이동 화살표 추가
6. 타임라인 사건과 연결
7. 미리보기 확인 후 저장

이미지 파일 업로드 시 원본 파일과 웹 최적화 파일을 구분해 보관한다. 원본 지도 위의 사용자 추가 표식은 이미지에 합성하지 않고 별도 좌표 데이터로 저장해 나중에 수정·삭제할 수 있게 한다.

### 27.9 수정 기능

- 폼을 통한 제목·날짜·지역·설명 수정
- 드래그로 사건 위치를 옮기되 저장 전 날짜와 지역 확인
- 기간선의 시작·종료 핸들을 드래그해 범위 수정
- 관계선의 출발·도착 지점 변경
- 지도 표식과 라벨 위치 이동
- 지도 파일 또는 URL 교체
- 수정 내역 미리보기
- 저장 전 유효성 검사

화면에서 위치를 이동하더라도 정식 데이터는 픽셀 좌표가 아니라 날짜, 지역 ID, 지도 좌표로 저장한다.

### 27.10 삭제 기능과 안전장치

- 삭제 전 대상 이름과 연결 관계 수를 확인
- 사건 삭제 시 연결된 관계선과 지도 표식을 함께 보여줌
- `사건만 삭제`, `연결도 함께 삭제` 등 영향 범위를 명확히 선택
- 즉시 영구 삭제보다 휴지통 또는 소프트 삭제 사용
- 삭제 직후 실행 취소 제공
- 일정 기간 후에만 영구 삭제
- 실수 복구를 위한 변경 이력 유지

출처 데이터는 다른 사건에서도 사용 중일 수 있으므로 참조 여부를 확인하지 않고 연쇄 삭제하지 않는다.

### 27.11 저장·게시 상태

편집 데이터에는 다음 상태를 둔다.

- 초안
- 검토 요청
- 게시
- 보관

편집 저장과 공개 게시를 분리한다. 수정한 내용은 초안으로 먼저 저장하고, 검토가 끝난 뒤 게시해야 일반 방문자에게 보인다. 단독 개인 프로젝트의 초기 단계에서는 초안과 게시만 구현하고 나머지는 추후 추가할 수 있다.

### 27.12 변경 이력

각 변경에는 다음을 기록한다.

- 변경 대상과 버전
- 변경 전·후 값
- 변경 시각
- 변경한 관리자
- 변경 이유 또는 메모
- 게시 여부

최소한 이전 버전 보기와 복원 기능을 제공한다. 역사 정보는 단순 UI 데이터보다 검증 과정이 중요하므로 누가 어떤 근거로 수정했는지 추적 가능해야 한다.

### 27.13 편집 권한

향후 여러 사용자가 참여할 경우 역할을 구분한다.

- 관리자: 모든 추가·수정·삭제·게시
- 편집자: 초안 작성과 수정
- 검토자: 검토 의견과 게시 승인
- 방문자: 열람만 가능

첫 버전은 관리자 한 명만 지원해도 되지만 데이터 구조에는 작성자·수정자·게시 상태를 포함한다.

### 27.14 편집 기능 데이터 확장

모든 주요 데이터에 다음 공통 필드를 추가한다.

- `status`
- `createdAt`
- `createdBy`
- `updatedAt`
- `updatedBy`
- `deletedAt` 선택값
- `version`
- `changeNote`

지도 위 사용자 표식은 다음처럼 별도 관리한다.

- `mapAnnotationId`
- `mapSnapshotId`
- `type`: point, arrow, frontLine, area, label
- `geometry`
- `style`
- `label`
- `description`
- `relatedEventIds`
- `sourceIds`

### 27.15 편집 기능 구현 순서

1. 읽기 전용 데이터 구조를 저장 가능한 형태로 변경
2. 로컬 개발용 편집 모드 구현
3. 사건 포인트 추가·수정·소프트 삭제
4. 기간선과 관계선 편집
5. 지도 업로드 및 타임라인 연결
6. 지도 위 표식·화살표·전선선 편집
7. 우클릭 컨텍스트 메뉴 추가
8. `+ 추가`, `⋯`, 키보드 대체 메뉴 추가
9. 초안·게시 상태와 변경 이력 구현
10. 관리자 인증과 권한 적용

우클릭 메뉴부터 만들기보다 기본 CRUD 폼을 먼저 구현한 뒤 우클릭을 빠른 진입 수단으로 연결한다.

### 27.16 편집 기능 완료 기준

- 특정 지역과 시점에 사건 포인트를 추가할 수 있다.
- 기간선과 지역 간 관계 화살표를 추가할 수 있다.
- 설명과 출처를 작성하고 수정할 수 있다.
- 기존 역사 지도와 지도 표식을 추가·교체할 수 있다.
- 모든 대상에 수정과 안전한 삭제가 제공된다.
- 편집 모드에서 우클릭 메뉴가 열리고 브라우저 메뉴와 겹치지 않는다.
- 모바일에서 길게 누르면 대상에 맞는 컨텍스트 메뉴가 열린다.
- 우클릭이나 길게 누르기 없이도 도구 모음, 항목 메뉴, 키보드로 같은 기능을 실행할 수 있다.
- 삭제한 항목을 복구할 수 있다.
- 저장과 공개 게시가 분리된다.
- 변경 이력을 확인하고 이전 버전으로 복원할 수 있다.
