# 세계사 타임라인 디자인 가이드

이 문서는 다른 AI, Claude AI, Codex 또는 새로운 세션에서 작업하더라도 현재 승인된 첫 화면 디자인, 이미지 합성 방식, 전쟁 지도 시각화를 동일하게 유지하기 위한 필수 명세다.

## 1. 현재 승인된 디자인

사용자가 2026년 7월 19일 승인한 첫 화면 카드 디자인을 기준으로 한다.

핵심 구성:

- 사건 카드는 역사적 시작 시점이 오래된 순서로 배열한다.
- 각 전쟁 카드는 한 장의 임의 생성 이미지가 아니라 실제 전쟁 사진과 실제 역사 지도 두 장을 사용한다.
- 위쪽에는 전쟁을 대표하는 실제 사진을 배치한다.
- 아래쪽과 배경에는 해당 전쟁 시기의 세력·동맹·전선 지도를 배치한다.
- 사진은 아래로 갈수록 투명해지고, 아래에 깔린 지도가 자연스럽게 드러난다.
- 지도는 완전한 그레이스케일로 처리한다.
- 카드 제목과 설명을 읽을 수 있도록 최종 어두운 오버레이를 추가하되 지도를 가리지 않을 정도로 유지한다.

이 구조를 이후 추가되는 모든 전쟁 카드의 기본 템플릿으로 사용한다.

## 2. 절대 피해야 할 방식

- AI 또는 개발자가 국가·대륙 윤곽을 임의로 그리지 않는다.
- 한반도, 유럽 등 실제 지형을 닮지 않은 추상 SVG를 지도처럼 사용하지 않는다.
- 역사적 세력 지도가 필요한 곳에 현대 백지도를 대신 사용하지 않는다.
- 외부 URL을 카드에서 직접 불러오지 않는다. 자료를 `images/`에 저장한 뒤 로컬 경로로 사용한다.
- 지도 한 장 또는 사진 한 장만으로 카드를 완성하지 않는다.
- 한국전쟁 지도 주변에 원본 이미지의 넓은 좌우 여백이 보이게 하지 않는다.
- 이미지 출처와 라이선스 기록을 생략하지 않는다.

## 3. 카드 레이어 순서

뒤에서 앞으로 다음 순서로 겹친다.

1. 역사 지도 `.war-map`
2. 대표 전쟁 사진 `.war-photo`
3. 사진·지도 통합용 미세 명암 레이어 `.war-composite::after`
4. 카드 전체 가독성 오버레이 `.card-overlay`
5. 연도, 상태, 제목, 설명, 이동 문구

권장 HTML 구조:

```html
<div class="card-art war-composite {topic-class}" role="img" aria-label="지도와 사진을 설명하는 문장">
  <img class="war-map" src="images/{map-file}" alt="">
  <img class="war-photo" src="images/{photo-file}" alt="">
</div>
<div class="card-overlay"></div>
```

두 이미지의 개별 `alt`는 비워 중복 낭독을 방지하고, 부모의 `aria-label`에서 두 자료의 의미를 한 번에 설명한다.

## 4. 현재 CSS 기준값

다른 AI가 레이아웃을 다시 만들더라도 다음 값을 출발점으로 사용한다.

### 지도

```css
.war-composite .war-map {
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  object-position: center;
  filter: grayscale(1) brightness(.72) contrast(1.34);
  opacity: 1;
  transform: scale(1.04);
}
```

### 사진

```css
.war-composite .war-photo {
  inset: 0 0 auto;
  width: 100%;
  height: 69%;
  object-fit: cover;
  object-position: center;
  filter: grayscale(.72) sepia(.16) brightness(.82) contrast(1.12);
  -webkit-mask-image: linear-gradient(
    to bottom,
    #000 0%,
    #000 56%,
    rgba(0, 0, 0, .82) 72%,
    transparent 100%
  );
  mask-image: linear-gradient(
    to bottom,
    #000 0%,
    #000 56%,
    rgba(0, 0, 0, .82) 72%,
    transparent 100%
  );
}
```

### 공통 오버레이

```css
.card-overlay {
  background:
    linear-gradient(
      180deg,
      rgba(13, 16, 14, .08),
      rgba(13, 16, 14, .11) 38%,
      rgba(13, 16, 14, .82) 100%
    ),
    linear-gradient(90deg, rgba(13, 16, 14, .2), transparent);
}
```

이 값은 현재 사용자가 `너무 좋다`고 승인한 결과다. 명확한 요청 없이 대폭 변경하지 않는다.

```css
.war-composite.korean-war .war-map {
  object-fit: contain;
  object-position: center 66%;
  padding: 6% 0 0;
  background: #354047;
  transform: scale(1.58);
}
```

한국 지도 확대값 `1.58`은 현재 승인된 값이다. 카드 비율을 변경하면 결과를 직접 확인한 뒤, 한반도 핵심 지역이 잘리지 않는 범위에서만 조정한다.

## 6. 새로운 전쟁 카드 추가 절차

1. 전쟁의 시작 연도와 종료 연도를 확인한다.
2. 해당 전쟁을 대표하는 실제 사진을 찾는다.
3. 해당 전쟁 시기의 세력·동맹·점령지·전선을 보여주는 지도를 찾는다.
4. 라이선스와 원출처를 확인한다.
5. 두 파일을 `images/`에 내려받는다.
6. `images/README.md`에 파일명, 제목, 제작자, 날짜, 라이선스, 원본 URL을 기록한다.
7. 카드에 `war-composite` 구조를 적용한다.
8. 지도는 그레이스케일로 유지한다.
9. 사진은 상단에 배치하고 아래로 갈수록 지도가 드러나게 한다.
10. 카드의 `data-start-year`를 입력해 시간순 자동 정렬에 포함한다.
11. 데스크톱과 모바일에서 사진의 핵심 피사체, 지도 윤곽, 카드 제목이 모두 보이는지 확인한다.

## 7. 일반 역사 사건 카드

전쟁이 아닌 사건에도 같은 원리를 응용하되 억지로 세력 지도를 사용하지 않는다.

- 혁명: 대표 사진·회화 + 당시 도시·국가 지도
- 제국의 성립·붕괴: 대표 유물·인물·건축 + 당시 세력 지도
- 탐험: 인물·선박 + 항로 지도
- 종교·문화 확산: 대표 유물·장소 + 확산 경로 지도
- 경제위기: 대표 사진·자료 + 영향 지역 또는 경제 흐름 지도
- 재난: 대표 사진 + 피해 범위 지도

즉, `대표 시각 자료 + 공간적 맥락 자료`의 두 레이어 원칙을 유지한다.

## 8. 정렬 규칙

- 첫 화면 카드는 `data-start-year`를 기준으로 오름차순 정렬한다.
- 고대 사건은 음수 연도를 사용한다.
- 예: 제2차 포에니 전쟁은 `data-start-year="-218"`.
- 중요도, 제작 순서, 게시 순서 때문에 시간순을 바꾸지 않는다.
- 동시대 사건의 세부 정렬은 정확한 시작일, 시작 월, 내부 정렬값 순서로 결정한다.

## 9. 현재 원본 파일

- 구현: `index.html`
- 전체 계획: `PLAN.md`
- 이미지 및 지도: `images/`
- 이미지 출처: `images/README.md`
- 디자인 재현 명세: `DESIGN_GUIDE.md`

새로운 AI는 첫 화면을 수정하기 전에 반드시 `PLAN.md`, `DESIGN_GUIDE.md`, `images/README.md`, `index.html`을 먼저 확인한다.

## 10. 변경 원칙

- 사용자의 명시적 요청 없이 승인된 두 이미지 합성 구조를 단일 이미지로 바꾸지 않는다.
- 이미지 교체 시에도 사진+지도 조합을 유지한다.
- 디자인을 변경한 뒤에는 카드 3개 모두에 일관되게 적용한다.
- 지도 가독성, 사진 대표성, 제목 가독성 순서로 검토한다.
- 지형이나 국경을 직접 임의 제작하지 않는다.
- 실제 자료가 없으면 임의로 만들지 말고 자료 탐색 또는 사용자 확인을 먼저 진행한다.

## 11. 전쟁 지도 공통 기준

### 11.1 기준 화면과 적용 범위

- 기준 구현은 `maps/ww2-theater-maps.html`의 1944년 이탈리아 전역 표현이다.
- 2026년 7월 21일 승인된 현재 화면의 시각 문법을 모든 전쟁 지도에 적용한다.
- 제2차 세계대전뿐 아니라 제1차 세계대전, 베트남 전쟁, 한국전쟁 등 이후 추가하는 지도도 같은 공통 렌더러를 사용한다.
- 특정 전역만을 위한 복제 코드나 좌표별 예외 CSS를 늘리지 않는다. 공통 기능과 사건 데이터를 분리한다.
- 이탈리아 화면은 기능·표현의 기준이다. 현재 좌표가 `개략`이면 역사적 정밀 경계의 근거로 재사용하지 않는다.

### 11.2 지도 경계와 투영

- 국가·대륙·해안선은 Natural Earth 같은 공개된 실제 경계 데이터를 사용한다.
- 기본은 Natural Earth 1:110m이며 좁은 전역의 정밀 경계는 1:50m 또는 1:10m를 사용한다.
- D3 지리 투영과 `geoPath`로 SVG에 렌더링한다.
- 국경·해안선을 임의의 다각형으로 대신 그리지 않는다.
- 현대 국경 데이터 사용 시 화면에 `현대 국경 기준`을 표시하고 출처도 밝힌다.
- 역사적 통제 범위와 전선이 추정치이면 반드시 `개략`으로 표시한다.
- 정밀 경계를 주장하려면 날짜, 자료 출처, 좌표 근거를 함께 저장한다.
- GeoJSON 폴리곤 winding 문제로 국가가 뒤집히거나 세계 반대편이 채워지지 않는지 브라우저에서 검증한다.

## 12. 공통 구현과 데이터 계약

### 12.1 권장 파일 구조

```text
assets/
  war-map-renderer.js       # 지도·영역·전선·경로·아이콘·라벨 공통 렌더러
  war-map.css               # 공통 색상, 선, 범례 스타일
data/
  ww2-map-events.js         # 제2차 세계대전 사건 데이터
  ww1-map-events.js         # 이후 전쟁별 데이터
maps/
  ww2-theater-maps.html     # 화면 구성과 데이터 선택
```

- 새 지도는 공통 렌더러를 호출하고 사건 데이터만 추가한다.
- 화면별 조건은 투영 범위, 범례 위치, 라벨 금지 영역 같은 설정으로 전달한다.
- 전선, 이동선, 아이콘, 통제 영역, 라벨 충돌 처리 코드를 전역마다 다시 만들지 않는다.

### 12.2 새 사건의 권장 형식

기존 배열은 호환을 위해 읽을 수 있게 하되 새 데이터는 객체로 작성한다.

```js
{
  date: "1944-05-18",
  text: "연합군이 구스타프선을 돌파",
  routes: [{
    from: [경도, 위도], to: [경도, 위도], label: "연합군 진격",
    side: "allied", status: "current",
    accuracy: "approximate", source: "출처 또는 설명"
  }],
  territories: [{
    kind: "allied-control", label: "연합군 통제(개략)",
    coordinates: [[경도, 위도], ...], labelAt: [경도, 위도],
    showLabel: true, accuracy: "approximate", source: "출처 또는 설명"
  }],
  zones: [{
    kind: "beachhead", label: "안치오 교두보",
    coordinates: [[경도, 위도], ...], labelAt: [경도, 위도], showLabel: true
  }],
  fronts: [{
    label: "구스타프선", status: "current",
    coordinates: [[경도, 위도], ...], labelAt: [경도, 위도],
    showLabel: true, accuracy: "approximate", source: "출처 또는 설명"
  }],
  retreats: [{
    from: [경도, 위도], to: [경도, 위도],
    label: "독일군 후퇴 방향", side: "axis"
  }],
  units: [{
    type: "tank", side: "allied", at: [경도, 위도],
    label: "기갑", heading: 25, showLabel: false
  }]
}
```

### 12.3 필드 규칙

- `side`: `allied`, `axis`, `neutral` 중 하나.
- `status`: `current`, `past`, `planned` 등 시간 상태.
- `accuracy`: `approximate` 또는 `source-based`.
- `source`: 정밀 표현에는 필수이며 개략 표현도 가능한 한 기록한다.
- `labelAt`: 선호 위치이며 충돌 계산이 최종 위치를 옮길 수 있다.
- `showLabel`: 혼잡한 이전 전선이나 보조 아이콘 라벨을 숨길 때 사용한다.
- 좌표는 항상 `[경도, 위도]` 순서다.

## 13. 진영 색과 통제 영역

### 13.1 색상

```css
--allied: #2f98f5;
--axis: #ff842b;
--neutral: #777;
--front: #202124;
--past-front: #9299a1;
```

- 연합군은 파랑, 추축군·독일군은 주황, 중립은 회색을 쓴다.
- 통제 영역의 권장 색 혼합 비율은 연합군 28%, 추축군 24%, 중립 17%다.
- 지형과 국경이 읽히도록 채움 불투명도를 높이지 않는다.
- 색상만 쓰지 않고 점, 빗금, 점선, 실선 같은 형태 차이를 함께 쓴다.

### 13.2 통제 영역 산정

- `통제 영역`은 해당 날짜에 실제로 통제한 육지다. 공격 예정지나 이동 경로를 통제 영역으로 칠하지 않는다.
- 국가 전체에 적용한 진영색은 `참전국·소속 진영`일 뿐 `점령지`가 아니다. 범례에서 두 개념을 반드시 분리한다.
- 한 국가가 분할 점령되면 전쟁 당시의 공개 역사 경계 GeoJSON을 별도 레이어로 사용하고, 점령 세력별로 내부를 나눠 칠한다.
- 폴란드처럼 현재 국경과 당시 국경이 크게 다르면 현대 폴란드 도형을 반으로 자르지 않는다. 전쟁 직전 역사 경계를 사용한다.
- 내부 분할선이 강·전선 등 문헌상의 기준을 단순화한 경우 지도와 범례에 `개략`을 표시하고 기준 날짜를 적는다.
- 국가 전체를 덮는 사각형이나 근거 없는 거대한 다각형을 사용하지 않는다.
- 실제 국가 GeoJSON을 클리핑 마스크로 쓰고 그 내부를 현재 전선을 기준으로 나눈다.
- 이탈리아처럼 연합군이 남쪽에서 북진하면 현재 전선 남쪽은 연합군, 북쪽은 독일군 통제로 표시한다.
- 해안선·국경은 실제 국가 형상을 따르고 전선 부근만 해당 날짜의 개략 전선으로 나눈다.
- 돌출부, 고립지, 교두보는 별도 보정 영역으로 추가한다.
- 근거가 개략적이면 `연합군 통제(개략)`, `독일군 통제(개략)`으로 쓴다. `장악`은 근거가 확실할 때만 쓴다.

### 13.3 교두보와 포위

- 교두보·고립지는 파란 점 패턴과 파란 외곽선으로 표시한다.
- 포위 압박은 교두보 위에 채움을 겹치지 않고 바깥을 감싸는 주황 점선의 닫힌 선으로 표시한다.
- 포위 압박선은 `fill: none`, `stroke-dasharray: 3 2`를 기준으로 한다.
- 범례도 `교두보·고립지`와 `포위 압박선`으로 분리한다.

## 14. 전선

### 14.1 현재 전선

- 짙은 회색의 얇은 실선이며 권장 굵기는 `0.9` SVG 단위다.
- 선을 따라 직각의 짧은 표식을 반복해 이동선과 구별한다.
- 표식 간격은 약 `12`, 중심선 양쪽 반길이는 각 `3` SVG 단위다.
- 경로 접선 벡터에 수직인 방향으로 표식을 계산한다.

### 14.2 이전 전선

- 회색 점선이며 권장 굵기 `0.65`, 점선 `3 3`, 불투명도 `0.62`다.
- 직각 표식 반길이는 약 `2.3`이다.
- 혼잡하면 이전 전선의 이름을 숨기고 범례만 남길 수 있다.

### 14.3 직각 표식 계산

```js
const total = pathNode.getTotalLength();
for (let d = spacing; d < total; d += spacing) {
  const p0 = pathNode.getPointAtLength(Math.max(0, d - 0.5));
  const p1 = pathNode.getPointAtLength(Math.min(total, d + 0.5));
  const dx = p1.x - p0.x;
  const dy = p1.y - p0.y;
  const len = Math.hypot(dx, dy) || 1;
  const nx = -dy / len;
  const ny = dx / len;
  // 중심점에서 ±tickHalfLength를 연결한다.
}
```

- 전선과 표식에는 `vector-effect: non-scaling-stroke`를 적용한다.
- 현재 전선 라벨은 이전 전선보다 높은 배치 우선순위를 갖는다.

## 15. 이동·후퇴 경로

- 연합군 이동선은 파랑, 추축군 이동선은 주황이다.
- 좁은 이탈리아 전역의 현재 이동선은 `1.1`, 과거 이동선은 `0.6`을 기준으로 한다.
- 넓은 전역은 현재 `3`, 과거 `1.5`까지 허용하되 화면상 굵기가 과도하지 않아야 한다.
- 시작점에는 진영색의 작은 원, 도착점에는 같은 색의 SVG 화살표 마커를 둔다.
- 마커가 있으면 큰 삼각형 도착 표식을 중복해서 그리지 않는다.
- 화살표는 진행 방향을 뜻하며, 주요 이동 방향의 개략 표현임을 밝힌다.
- 독일군 후퇴는 별도 데이터로 관리하고 주황 점선과 주황 화살표로 표시한다.
- 후퇴선 권장 굵기는 `1.4`, 점선은 `4 3`이며 범례에 `독일군 후퇴 방향`을 넣는다.
- 선에는 `vector-effect: non-scaling-stroke`를 적용한다.
- 줌 배율이 `k`이면 아이콘·고정 표식 그룹에 `scale(1 / k)`를 적용해 화면 기준 크기를 유지할 수 있다.

## 16. 공격 수단 아이콘

### 16.1 공통 심벌과 크기

- 운영체제 이모지에 의존하지 않고 공통 SVG `<symbol>`을 사용한다.
- 필수 ID는 `unit-tank`, `unit-ship`, `unit-bomber`, `unit-landing`이다.
- 지도 기본 크기는 약 `18 × 13`, 범례는 약 `17 × 12` SVG 단위다.
- 확대해도 거대해지지 않도록 화면 기준 크기를 유지한다.
- 연합군 아이콘은 파랑, 추축군은 주황, 미지정은 짙은 회색이다.
- 진영 색상 키를 범례에도 반드시 넣는다.

### 16.2 식별 가능한 세부 묘사

- 전차: 무한궤도, 최소 4개 보기륜, 차체, 포탑, 해치, 포신.
- 함대: 선체, 갑판, 상부 구조물, 마스트, 함포.
- 폭격기: 동체, 주익, 엔진, 꼬리날개.
- 상륙정: 선체, 내부 갑판 리브, 함수 램프, 조타실.
- 단순 사각형이나 식별하기 어려운 기호로 대체하지 않는다.
- `heading`이 있으면 시각 중심을 기준으로 회전해 진행 방향을 표시한다.
- 좁은 지도에서는 개별 아이콘 라벨을 숨기고 범례로 식별한다.

## 17. SVG 레이어 순서

1. 바다 배경과 경위선
2. 국가·해안선·국경선
3. 연합군·추축군·중립 통제 영역
4. 교두보·고립지·포위 압박선
5. 현재·이전 전선과 직각 표식
6. 후퇴 경로
7. 과거 이동 경로
8. 현재 이동 경로와 방향 화살표
9. 시작점·도착점 표식
10. 전차·함대·폭격기·상륙정 아이콘
11. 라벨 연결선
12. 모든 지도 라벨
13. 범례

- 모든 글자는 영역, 선, 아이콘보다 위에 있어야 한다.
- 글자를 각 도형 그룹에서 바로 그리지 않는다. 라벨 요청을 수집해 마지막 전용 레이어에서 한꺼번에 배치한다.
- 라벨 레이어는 `pointer-events: none`으로 지도 조작을 방해하지 않게 한다.
- 범례는 최상위에 두되 라벨 계산 전에 범례 전체를 금지 영역으로 등록한다.

## 18. 라벨 충돌 방지

### 18.1 필수 원칙

- 글자가 선·영역·아이콘에 가려져서는 안 되며 글자끼리 겹쳐서도 안 된다.
- 실제 SVG `getBBox()`로 글자 크기를 측정한 뒤 배치한다.
- 데이터의 기준점을 우선하되 충돌하면 가장 가까운 안전 위치로 옮긴다.
- 도형을 모두 그린 다음 라벨을 계산하며 화면 크기·투영·줌·사건이 바뀌면 다시 계산한다.

### 18.2 후보 위치와 점수

최소 다음 후보를 검사한다.

```js
[
  [0, 0],
  [8, 0], [-8, 0], [0, 8], [0, -8],
  [8, 8], [-8, 8], [8, -8], [-8, -8],
  [16, 0], [-16, 0], [0, 14], [0, -14],
  [24, 16], [-24, 16], [24, -16], [-24, -16]
]
```

- 지도 밖 후보는 제외한다.
- 이미 배치한 글자, 아이콘 장애물, 범례 금지 영역과의 겹침 면적을 계산한다.
- 점수는 `겹침 면적의 큰 가중치 + 원래 위치에서 이동한 거리`로 계산한다.
- 무충돌 후보 중 가장 가까운 위치를 선택한다. 없으면 총 겹침 점수가 가장 낮은 후보를 선택한다.
- 기준점에서 `10` SVG 단위 이상 이동하면 기준점과 글자를 잇는 얇은 연결선을 그린다.
- 아이콘은 중심점 기준 좌우 `11`, 상하 `9` 정도를 장애물 상자로 등록한다.

### 18.3 라벨 우선순위

| 라벨 | 우선순위 |
|---|---:|
| 사건 단계·날짜 | 110 |
| 현재 전선 | 100 |
| 교두보·고립지 | 88 |
| 후퇴 방향 | 82 |
| 이동 경로 도착지 | 78 |
| 이동 경로 출발지 | 76 |
| 통제 영역 | 72 |
| 이전 전선 | 62 |
| 개별 부대·공격 수단 | 54 |

- 높은 우선순위부터 배치한다.
- 낮은 우선순위의 보조 라벨은 공간이 없으면 숨길 수 있다.
- 사건 날짜와 현재 전선처럼 필수 정보는 숨기지 않는다.

### 18.4 범례 금지 영역

- 이탈리아 기준 범례 영역은 대략 다음과 같으며 실제 구현에서는 범례 `getBBox()`를 측정한다.

```js
{ x1: width - 188, y1: 18, x2: width - 18, y2: 284 }
```

### 18.5 권장 처리 순서

```js
const labels = collectAllLabels().sort((a, b) => b.priority - a.priority);
const obstacles = collectIconBoxes();
const reserved = [measureLegendBox()];
labels.forEach(label => placeCollisionAwareLabel({
  label, obstacles, reserved, alreadyPlaced
}));
drawLabelConnectors();
drawLabelTextOnTop();
```

## 19. 범례

- 기본 위치는 오른쪽 위다. 이탈리아 기준 `translate(width - 188, 18)`, 약 `170 × 266`이다.
- 작은 화면에서는 접기, 반투명도 조절 또는 지도 아래 배치를 허용한다.
- 실제 사용한 항목만 넣고 지도와 동일한 색·패턴·선·SVG 심벌을 재사용한다.
- 사용 시 포함할 항목: 연합군 통제, 독일군 통제, 중립, 교두보·고립지, 포위 압박선, 현재 전선, 이전 전선, 독일군 후퇴 방향.
- 공격 수단 항목: 전차, 함대, 폭격기, 상륙정.
- 진영 키: 연합군 아이콘 색, 추축군 아이콘 색.
- `화살표는 진행 방향`이라는 설명을 넣는다.
- 범례 패널은 옅은 반투명 배경과 얇은 테두리를 사용한다.

## 20. 확대·축소와 패닝

- `+`, `−` 버튼, 마우스 휠, 트랙패드, 터치 핀치를 지원한다.
- 드래그와 터치로 패닝할 수 있어야 한다.
- 경로 추가·편집 중에도 확대·축소와 패닝이 작동해야 한다.
- 사건에는 지도 중심점과 확대 배율을 저장해 카드 미리보기에서도 같은 구도를 재현한다.
- 추가한 요소가 뷰포트 밖이면 자동 맞춤하거나 저장된 뷰로 해당 요소를 보이게 한다.
- 확대 후에도 선, 전선 표식, 화살표 끝, 아이콘, 라벨이 과도하게 커지지 않게 보정한다.

## 21. 반응형과 접근성

- SVG는 `viewBox`와 비율 유지 설정을 사용한다.
- 데스크톱 Safari·Chrome과 모바일 Safari에서 확인한다.
- 다크 모드에서 국가가 검은 덩어리로 보이지 않도록 국가·바다 채움색을 명시한다.
- SVG에 `<title>`과 `<desc>`, 조작 버튼에 접근 가능한 이름을 제공한다.
- 색상 외에도 패턴, 선 종류, 아이콘 형태로 의미를 구별한다.
- 키보드로 사건 선택과 확대·축소가 가능해야 한다.

## 22. 제작 절차

1. 사건의 날짜와 표현할 의미를 확정한다.
2. 실제 공개 지도 데이터와 전역에 맞는 축척을 선택한다.
3. 투영과 초기 화면 영역을 정한다.
4. 신뢰할 수 있는 자료에서 전선·통제 지역·이동 방향을 확인한다.
5. `accuracy`와 `source`를 기록한다.
6. 통제 영역을 실제 육지 형상에 클리핑한다.
7. 현재·이전 전선과 직각 표식을 그린다.
8. 이동·후퇴 경로와 화살표를 그린다.
9. 진영 색이 적용된 공격 수단 아이콘을 배치한다.
10. 모든 라벨을 수집해 충돌 방지 알고리즘으로 배치한다.
11. 실제 사용한 표현만 범례에 넣는다.
12. 사건 전환, 확대·축소, 패닝을 실제 브라우저에서 검증한다.

## 24. 금지 사항

- 임의의 국경·해안선을 실제 지도처럼 사용하지 않는다.
- 통제 영역을 근거 없는 큰 사각형이나 국가 밖 다각형으로 표시하지 않는다.
- 좁은 지도에 굵은 이동선, 큰 화살표, 큰 아이콘을 놓지 않는다.
- 포위 압박과 교두보를 동일 위치의 불투명 채움으로 겹치지 않는다.
- 글자를 중간 레이어에 그려 이후 선이나 아이콘이 덮게 하지 않는다.
- 사건마다 라벨 좌표를 끝없이 수작업 조정하지 않는다. 공통 충돌 배치기를 우선한다.
- 아이콘을 진영 구분 없이 모두 검정으로 표시하지 않는다.
- 범례와 실제 기호의 색, 선, 패턴, 아이콘을 다르게 만들지 않는다.
- 개략 표현을 정확한 점령 경계라고 단정하지 않는다.
- 실제 브라우저에서 검증하지 않은 지도를 완료로 처리하지 않는다.
