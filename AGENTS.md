# AI 작업 지침

이 파일의 규칙은 이 저장소에서 작업하는 모든 AI 도구와 세션에 적용한다.

## 필수 문서 확인

- 작업을 시작하기 전에 `PLAN.md`를 읽는다.
- 첫 화면 또는 사건 카드를 수정할 때는 `DESIGN_GUIDE.md`를 반드시 읽고 승인된 디자인을 유지한다.
- 이미지를 추가하거나 교체할 때는 `images/README.md`의 출처 기록 형식을 따른다.

## Git 커밋 규칙

- 사용자가 커밋을 요청하면 일반 개발자 커밋처럼 작성한다.
- 커밋 메시지 제목과 본문에 AI로 작업했다는 표현을 넣지 않는다.
- `Co-authored-by`, `Co-worked-by`, `Generated-by` 등 AI 도구 또는 AI 공동작업자를 나타내는 트레일러를 절대 추가하지 않는다.
- `Codex`, `Claude`, `ChatGPT`, `AI` 등의 이름을 커밋 메시지에 자동으로 넣지 않는다.
- 커밋 작성자 또는 공동 작성자를 AI 이름으로 변경하지 않는다.
- 별도 요청이 없으면 간결하고 작업 내용을 직접 설명하는 커밋 메시지를 사용한다.

## 현재 승인된 첫 화면 원칙

- 사건 카드는 시작 시점이 오래된 순서로 정렬한다.
- 전쟁 카드에는 실제 대표 사진과 실제 세력·전선 지도 두 장을 사용한다.
- 위에는 전쟁 사진, 아래에는 그레이스케일 지도를 배치하고 그라데이션으로 합성한다.
- 국가·대륙·한반도 윤곽을 임의로 그려 지도처럼 사용하지 않는다.
- 외부 자료는 `images/`에 저장하고 출처를 `images/README.md`에 기록한다.
- 세부 CSS 값과 전쟁별 이미지 조합은 `DESIGN_GUIDE.md`를 기준으로 한다.

## 전쟁별 시기 지도 원칙

- 각 전쟁의 상세 화면에서는 주요 시기를 시간순으로 선택해 볼 수 있는 인터랙티브 지도를 제공한다.
- 전쟁 상세 화면의 기본 구조는 모든 지역·전선을 한 화면에 동시에 나열하는 가로 열 구조로 한다.
- 각 지역 열에는 위에서 아래로 흐르는 세로 타임라인을 두고, 각 사건은 짧은 가로선으로 날짜·사건 카드와 연결한다.
- 지도화할 수 있는 사건은 카드 오른쪽에 실제 지도 데이터로 만든 축소 지도를 표시하고, 누르면 같은 지도를 큰 화면으로 확대한다.
- 모바일에서는 네 지역 열을 없애거나 탭으로 숨기지 말고 가로 스크롤로 모두 유지한다.
- 지도 구현은 `maps/ww2-theater-maps.html`과 같은 방식을 기본으로 한다.
- 실제 공개 지도 경계 데이터(예: Natural Earth GeoJSON/TopoJSON)를 D3 지리 투영으로 SVG에 렌더링한다.
- 국경선과 지형 윤곽을 임의로 손그림처럼 만들지 않는다.
- 전쟁에 여러 전선이나 지역이 있으면 지도 또는 탭을 전선·지역별로 구분한다.
- 각 시기에는 연도·월, 핵심 사건 설명, 주요 진격·철수·상륙·해상 이동 경로를 화살표로 표시한다.
- 이전/다음 버튼과 시기 선택 기능을 제공하고, 앞선 사건의 경로와 현재 사건의 경로를 구별한다.
- 역사적 이동 경로는 주요 방향을 개략적으로 보여주는 것임을 명시한다.
- 현대 국경 데이터를 사용하면 현대 국경 기준임을 밝히고 지도 데이터 출처를 표시한다.
- 모바일과 맥북의 일반 브라우저에서 모두 단독 실행되도록 필요한 CSS 변수와 기본 스타일을 HTML에 포함한다.
- 완성 전 모든 지도 탭, 시기 선택, 투영, 지명, 화살표와 밝은 화면·어두운 화면 표시를 실제 브라우저에서 검증한다.
